Java 概述

发表于2024-12-22|更新于2024-12-25|Java
|浏览量:

什么是程序

  • 程序：计算机执行某些操作或解决某个问题而编写的一系列有序指令的集合

  • 体验

Java 诞生小故事

  • 1990 Sun 公司启动绿色计划

  • 1992 创建 oak（橡树）语言 -> Java

  • 1994 gosling 参加硅谷大会演示 Java 功能，震惊世界

  • 1995 Sun 公司正式发布 Java 的第一个版本

  • 2009 甲骨文公司收购 Sun 公司

  • 2011 发布 Java 7

Java 技术体系

  • Java SE 标准版

    支持面向桌面应用

  • Java EE 企业版

    为开发企业环境下的应用程序提供一套解决方案

  • Java ME 小型版

    支持 Java 程序运行在移动终端上的平台

Java 特点

  1. Java 语言是面向对象的（OOP）

  2. Java 语言是健壮的，Java 的强类型机制、异常处理、垃圾的自动回收等是 Java 程序健壮性的重要保障

  3. Java 语言是跨平台性的

  4. Java 语言是解释性的

Java 的开发工具

  • editplus

  • Notepad++

  • Sublime Test（前期使用）

  • IDEA（后期使用）

  • eclipse（后期使用）

什么是 JDK、JRE 和 JVM

Java 程序运行在一个名为 JVM 的虚拟机中，每个平台有每个平台对应的虚拟机，所以一套 Java 程序可以运行在不同的平台上

JVM

  • Java 核心机制-Java 虚拟机（Java virtual mechine）

    1. JVM 是一个虚拟的计算机，具有指令集并使用不同的存储区域，负责执行指令、管理数据、内存、计算器，包含在 JDK 中

    2. 对于不同平台，有不同的虚拟机

    3. Java 虚拟机的机制屏蔽了底层运行平台的差别，实现了”一次编译，到处运行“

JDK

  1. JDK 的全程是：Java Development Kit

  2. JDK = JRE + Java 开发工具（java、javac、javadoc 等）

  3. JDK 是提供给 Java 开发人员使用的，其中包含了 Java 的开发工具和 JRE

JRE

  1. JRE 的全称是：Java Runtime Environment

  2. JRE = JVM + Java 核心类库

  3. 如果只是想运行 Java 程序，只需要安装 JRE 即可

JVM、JDK、JRE 三者关系

  1. JDK = JRE + Java 开发工具

  2. JRE = JVM + Java 核心类库

  3. JDK = JVM + Java 核心类库 + Java 开发工具

下载安装 JDK

Java Downloads | Oracle 中国

根据自己电脑的平台去下载对应平台的 JDK，其中 X86 为 32 位系统，X64 为 64 位系统

安装时，切记不要安装在含有中文的路径下

在安装 JDK 时，会同步安装 JRE

配置环境变量

通过配置环境变量，将 Java 命令添加至环境变量中，我们就可以任意路径下使用 Java 命令了

  1. 此电脑 → 属性 → 高级系统设置 → 环境变量

  2. 在系统变量下点击新建，变量名：JAVA_HOME，变量值：Java 的安装路径

  3. 再找到系统变量中的 Path 环境变量，点击编辑，点击右侧的新建，将 %JAVA_HOME\bin% 添加至其中

Java 快速入门

需求说明

开发一个 HelloWorld 程序，可以输出 ”Hello, World!“

开发步骤

  1. 创建名为 HelloWorld.java 的文件

  2. 在文件内编写代码

  1. 使用 javac HelloWorld.java 命令将 java 文件编译为可执行的 class 文件，再使用 java HelloWorld 命令执行程序

注意

-控制台的编码格式需要与文件的编码格式一致

课堂小练习

  • 开发一个程序，可以输出 “XXX is studying Java!‘

Java 开发快速入门

Java 执行流程分析

  • .java 文件 →（javac 编译）→ .class 文件 →（java 运行）→ 结果

什么是编译

  • 有了 Java 源文件，可以通过编译器将其编译成 JVM 可以识别的 字节码文件

  • 在源文件路径下，使用 javac 命令将 java 文件进行编译，生成 class 文件

  • 如果程序没有任何报错，则可以看到在当前目录下生成一个同名的 class 文件

什么是运行

  • 有了可执行的字节码文件，可以通过 java 命令执行字节码文件

  • 本质就是将字节码文件装载到 JVM 中，让 JVM 去执行

注意事项

  • 每次对 java 源文件进行修改之后，都需要重新编译，重新生成新的 class 文件，不然再次执行时，使用的是之前的字节码文件

Java 开发注意事项和细节说明

  1. Java 源文件以 .java 为扩展名，源文件的基本组成部分是类

  2. Java 程序的执行入口是 main 方法，该方法的写法是固定的

  3. Java 语言是严格区分大小写的

  4. Java 代码由语句构成，每一条语句必须以 ; 结尾

  5. 大括号是成对出现的，编写代码时不要分开写，要成对书写

  6. 一个源文件中可以存在多个类，但是有且仅有一个类可以被 public 修饰

  7. 源文件的文件名必须与被 public 修饰的类的类名一致

  8. main 方法不是必须写在被 public 修饰的类中，也可以写在同文件的其他类中

易犯错误

  1. 执行命令时，文件名写错

  2. 源文件名与类名不一致

  3. 缺少符号，比如缺少分号或者大括号

  4. 字符不是英文格式而是中文格式

  5. 代码单词拼写错误，比如：main 写成 mian

注释

说明

  • 用于对代码进行解释说明的文字，提高代码的可阅读性

注释类型

单行注释

多行注释

文档注释

注意

  • 被注释的文字，在编译时会被忽略

  • 多行注释不允许嵌套

Java 代码规范

  1. 类、方法的注释使用文档注释

  2. 单条语句或代码块使用单行注释或多行注释

  3. 使用 Tab 控制代码缩进而不是空格

  4. 运算符与变量之间使用空格隔开，如 int a = 10;

  5. 文件格式应为 UTF-8 编码

  6. 单行代码长度不超过80个字符

  7. 大括号的开头写在同一行，大括号的结尾写在单独的一行

DOS 常用指令

DOS 介绍

  • DOS：Disk Operating System，磁盘操作系统，是计算机的启动和运行环境，是 Windows 操作系统的核心，是 Windows 操作系统运行必须的。用户可以通过 DOS 命令行来管理计算机，包括文件管理、目录管理、系统管理、网络管理等。

路径介绍

  • 相对路径：相对于当前目录的路径

  • 绝对路径：相对于根目录的路径

DOS 命令

  1. dir ：查看当前目录下的文件和目录

  2. 盘符名 + 冒号 ：切换盘符

  3. cd + 路径 ：切换目录

  4. cd + .. ：返回上一级目录

  5. cd + . ：返回当前目录

  6. cd + / ：返回根目录

  7. mkdir + 目录名 ：创建目录

  8. rmdir + 目录名 ：删除目录

  9. copy + 源文件名 + 目标文件名 ：复制文件

  10. move + 源文件名 + 目标文件名 ：移动文件

  11. del + 文件名 ：删除文件

  12. ren + 源文件名 + 目标文件名 ：重命名文件
